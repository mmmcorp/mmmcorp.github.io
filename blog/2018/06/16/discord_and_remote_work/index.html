ゲーマー向けボイスアプリのDiscordがリモートワークと相性が良かった

Tweet

社員全員がフルでリモートワークをしている弊社では、コミュニケーションは基本的に Slack で文字のやり取りをして、音声通話では Skype を利用していた。
数ヶ月前に、 Discord を利用し始めたら意外とリモートワークとの相性が良いと感じたので、今回はその紹介をしたいと思う。

きっかけ

ある案件で、作業進捗がかなり厳しい状態になってしまい、チャットベースのコミュニケーションだけではなく、もっと密なコミュニケーションを取りながら、開発していく必要があった。
最初は、これまで使ってきた Skype を常時接続状態にして作業しようか、ということも検討したが、 Skype だと複数人で通話している際に、その通話のホストの人が抜けてしまうと、通話が終了してしまう懸念点もあったので、「ゲーマー向けボイス＆テキストチャットアプリ」の Discord を使ってみることになった。

良かった点

使ってみて良かった点としては、

  • 気軽に話しかけられる
  • 気軽にミーティングに参加できる

点かな、と感じた。

気軽に話しかけられる

込み入った実装や、議論が必要な設計に関する相談はもちろんのこと、これってどうやったらキレイに書けるんだろう、これってどうなってたっけ、というようなちょっと確認したいことがあったときに、チームメンバーにすぐに、気軽に話しかけられるのは、スピードが求められる案件等では非常に良いな、と感じた。
Slack & Skype だと、 Slack で

「ちょっと Skype いいですか？」

からの、 Skype で関係者にかける、というステップが必要だった。

気軽にミーティングに参加できる

これは、最近感じているメリットだが、 Discord を使って行うミーティングは、社内のメンバーには常にオープンになっているという点。
弊社では、基本的に社内のミーティングのスケジュールも Google Calendar で共有している。
会議招集されていないメンバーも、Discord を使って行うミーティングには、興味があったら、気軽に参加(傍聴)できる。
Skype を使ったミーティングでは、会議招集したメンバーのみに通話をかけて、ミーティングが進んでしまうため、仮に参加(傍聴)したくても、 Slack で

「参加していいですか？」
「通話に追加しますね」

という手間が発生する。
また Skype では、通話に参加していないメンバーには、まだ通話しているのか、もう終わっているのか、ということがわからなかったため、 Slack で

「まだミーティングしてます？」
「あ、ついさっき終わったところです」

といったやり取りが発生していた。
Discord では話をしていないときには、マイクをミュートにしておけば、他のメンバーにも話をしているのか否かということが共有できるため良かった。

また、

「フロントエンドチームにいるけど、バックエンド側の実装や議論も気になる」
「今はバックエンドチームにいるけど、今後フロントエンドの技術もやってみたいから興味あるし、聞いておきたい」
「直接あの案件には関わっていないけど、人手が足りてなさそうだし手伝えることがあるかもしれないから参加してみよう」

といった場合にも、常に会議がオープンな状態になっていて、気軽に参加できるのは、チーム間・プロジェクト間の垣根を下げる点、社内での情報共有と言った点で良いことだな、と感じた。

悪かった点

逆に、悪かった点(使う上で注意が必要な点)は

  • 画面共有ができない
  • 音声がたまに途切れる
  • Slack と Discord で議論が分散する

という点だった。

画面共有ができない

画面共有をしながらのオンラインレビューを行いたいとき、 Discord のチャンネル上では画面共有できない(ダイレクトメッセージを使えば可能)。
そのため画面共有が必要なときには、 Skype を使って話すことが多い。

音声がたまに途切れる

これは、参加者側の環境に問題があるのかもしれないが、たまに音声が途切れがちになったりする。
朝のミーティング時はちゃんと聞こえたのに。今はちょっと途切れがちになっている、ということがちょくちょくある。
設定をいじれば改善するのかもしれないが、まだ原因がわかっていない。
だいたい、 Discord の再起動等でちゃんと聞こえるようになることが多い。

Slack と Discord で議論が分散する

Discord には、テキストチャンネルもあり、チャットもできる。
Slack と併用している場合は、議論が分散してしまい、あとで見直すときなど大変になる。
そのため弊社では 文字のやり取りは Slack を使い、 Discord のテキストチャンネルは使わない運用とした。

まとめ

「ゲーマー向け」と謳っている Discord が意外とリモートワークをする上で便利だったので、紹介した。
他にもこんなツールやサービスが便利だった、ということがあれば随時紹介していきたい。
